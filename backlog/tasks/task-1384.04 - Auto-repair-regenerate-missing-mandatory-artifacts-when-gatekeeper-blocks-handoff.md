## Problem
The gatekeeper pre-review (handoff Step 2.5) blocks handoff when mandatory artifacts are missing (MISSION.md, CP-*.md checkpoints, nel-record, etc.). Today this is a hard block requiring a human to notice and re-run the agent.

## Fix
When gatekeeper reports specific missing artifacts, trigger an automatic agent relaunch with a targeted prompt to generate exactly the missing artifacts (reuse the CP-1 auto-generation scaffolding in `handoff.js` and the relaunch path in `active.js`). Bounded retry, then escalate.

## Acceptance
- Missing-artifact gatekeeper failure triggers a targeted regeneration prompt naming the missing files.
- Bounded retry; clean escalation on failure.
- Test covering the trigger.
