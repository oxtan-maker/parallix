## Description

## Problem
When a declared gate fails for a real reason (tests red, typecheck error, lint), `repair-handoff.js` classifies it as non-repairable (only dirty-tree and behind-remote are handled, `repair-handoff.js:105-127`) and strands the run. The human must manually prompt the agent to fix.

## Fix
Extend the repair path so a genuine gate failure relaunches the implementer agent (see existing relaunch scaffolding in `lib/commands/active.js:293,335,453`) with a structured fix prompt containing: the failing gate command, its stdout/stderr, and the instruction to fix and re-run that gate. Cap retries (e.g. 1-2) to avoid loops, then escalate to human.

## Acceptance
- A failing real gate triggers an automatic agent relaunch with the gate's captured output in the prompt.
- Bounded retry count; on exhaustion, clean escalation message (not a silent strand).
- Test covering the relaunch trigger and the retry cap.
