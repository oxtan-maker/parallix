## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
Convert the 6 simplest command handlers. These have fewer inter-command dependencies and lighter logic than the workflow/integration commands.

**Files renamed `.js` → `.ts`:**
- `lib/commands/config.js` (2 requires) — print effective configuration
- `lib/commands/diff.js` (5 requires) — branch-vs-main diff tool
- `lib/commands/status.js` (8 requires) — mission/repo overview
- `lib/commands/coverage-gate.js` (5 requires) — coverage gate enforcement
- `lib/commands/stats.js` (12 requires) — usage statistics from CSV
- `lib/commands/stats-backfill.js` (7 requires) — backfill stats data

**Conversion details:**
- Replace `require()` with ES `import` from converted modules in core/agents/review/tools
- Replace `module.exports` with ES `export`
- Preserve JSDoc annotations (these files already have good coverage)
- `stats.js` is the most complex here (12 requires) — it imports from core, agents, review, and tools

**No-committed-`.js` guardrails (apply the canonical version in TASK-1364) — directory `lib/commands`:**
- Faithful rename per file: `git diff -M --summary <merge-base> -- lib/commands/X.js lib/commands/X.ts` reports a `rename` ≥ 50%. Preserve import names, helpers, formatting, comments, and the exact export shape — do not rewrite or bloat JSDoc.
- After converting each file: `git rm --cached lib/commands/X.js` (regenerated by `npm run build:cjs`; no `package.json` change — it already compiles all `lib/**/*.ts`).
- `lib/commands` is converted across TASK-1369/1370/1371. **Whichever runs first** adds `lib/commands/*.js` to `.gitignore` and `.eslintignore` (with a `!lib/commands/<file>.js` negation for every still-hand-written command); each wave then **deletes the negation** for the files it converts, so `.eslintignore` always negates exactly the remaining hand-written commands.
- Gates (capture proof): `tsc --noEmit` clean; `npm test` at baseline; `git ls-files lib/commands/{config,diff,status,coverage-gate,stats,stats-backfill}.js` empty; rename shown per file; `grep -c 'module.exports' lib/commands/*.ts` zero; `./scripts/verify-local.sh static-analysis` green **with compiled `lib/commands/*.js` present**; `npm run prepublishOnly && npm pack --dry-run | grep 'lib/commands/'` shows compiled `.js`; `node -e "require('./lib/commands/config')"` loads with exports intact.
