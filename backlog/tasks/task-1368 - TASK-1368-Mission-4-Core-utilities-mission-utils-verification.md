## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
Convert the 2 core utility modules that handle mission lifecycle operations and verification gates. These are mid-sized files with good JSDoc coverage.

**Files renamed `.js` → `.ts`:**
- `lib/core/mission-utils.js` (1027 lines) — mission directory management, worktree ops, merge conflict detection, checkpoint handling. **Largest non-command file.**
- `lib/core/verification.js` (149 lines) — verification gate execution and reporting

**Conversion details:**
- Convert as a **faithful, minimal rename** (`git mv`-equivalent): keep import names, helper functions, formatting, blank lines, and comments — change only what ESM/TS requires. `mission-utils.js` is 1027 lines, so a rewrite would destroy git rename detection; aim for `git diff -M` similarity **≥ 50%**.
- Replace `require()` with ES `import` (relative imports use explicit `.js` extension); a minimal lazy `require()` is acceptable only to break a genuine circular dependency.
- Replace `module.exports` with ES `export`, preserving the exact public export shape (including any `_internals`).
- Add `@type`/inline annotations where JSDoc is present (these files have good JSDoc coverage)
- Define TypeScript interfaces for mission state, checkpoint structures, and verification results
- `mission-utils.js` has injectable dependencies (gitRunner, fsModule, pathModule) — preserve this pattern

**No autogenerated `.js` in git:** follow the canonical "no-committed-`.js`" guardrails in TASK-1364. This dir already has its `.gitignore`/`.eslintignore` `lib/core/*.js` globs and the `build:cjs` / `pretest` / `prepublishOnly` wiring from an integrated TASK-1367 — **no new infra is needed**. Concretely for this wave:
- `git rm --cached lib/core/mission-utils.js lib/core/verification.js` (delete on disk; regenerated by `npm run build:cjs`).
- In `.eslintignore`, **delete** the `!lib/core/mission-utils.js` and `!lib/core/verification.js` negation lines so their compiled output is ignored (the remaining `!lib/core/nels.js` and `!lib/core/subagent-limit.js` stay until those files are converted).

**Acceptance gates (in addition to the standard DoD):**
- [ ] `tsc --noEmit` clean and `npm test` at baseline pass counts
- [ ] `git ls-files lib/core/mission-utils.js lib/core/verification.js` — empty (no compiled `.js` tracked)
- [ ] `git diff -M --summary <merge-base> -- lib/core/mission-utils.js lib/core/mission-utils.ts | grep -q rename` (and same for `verification`) — history shows move + ts fixes
- [ ] `grep -c 'module.exports' lib/core/mission-utils.ts lib/core/verification.ts` — zero; `require(` only the permitted circular-dep break, if any
- [ ] `./scripts/verify-local.sh static-analysis` passes **with compiled `lib/core/*.js` present** on disk
- [ ] `npm run prepublishOnly && npm pack --dry-run | grep -E 'lib/core/(mission-utils|verification)\.js'` — compiled `.js` present in the package
- [ ] `node -e "require('./lib/core/mission-utils'); require('./lib/core/verification')"` loads without error, exports preserved

**Dependency:** Depends on TASK-1365 (infrastructure), TASK-1366 (core foundation), and TASK-1367 (storage/persistent-data-migration + the no-committed-`.js` / deterministic-publish infrastructure).
