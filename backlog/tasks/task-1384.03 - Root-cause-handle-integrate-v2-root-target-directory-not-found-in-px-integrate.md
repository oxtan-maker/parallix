## Problem
The outer `px integrate` step fails with:
```
[px] ERROR: target directory '/tmp/integrate-v2-root' not found.
```
This is not in this repo's `lib/`/`scripts/` — it lives in the outer px orchestrator. It recurs (seen in task-1348, task-1381, and the task-1372 run). When it fires after a successful handoff, the run is left half-done with no auto-recovery.

## Goal
- Find where `/tmp/integrate-v2-root` is created/expected in the px orchestrator.
- Decide correct behavior: create the directory on demand, skip the v2 integrate path gracefully, or surface a clear remediation rather than a raw ERROR + strand.
- Add a self-heal or skip-with-warning so a missing integrate root does not block an otherwise-complete handoff.

## Acceptance
- Root cause of the missing directory documented.
- `px integrate` no longer hard-fails the whole run on a missing integrate root — it either provisions it or skips cleanly with a logged reason.
- Linked back from task-1381 / task-1348.
