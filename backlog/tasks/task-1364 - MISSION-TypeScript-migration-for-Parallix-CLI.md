## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
Convert the entire Parallix codebase from JavaScript to TypeScript. This is a 23K-line, 62-file migration across 5 subsystems (core, commands, agents, review, tools) with 107 existing tests.

**Outcome:** Zero runtime dependencies, fully typed TypeScript codebase with passing tests, ESLint integration, and npm-publishable output.

**Constraints:**
- **`.ts` is the only source of truth in git. Never commit TypeScript-compiled `.js`.** Compiled `.js` are generated deterministically by `npm run build:cjs` (gitignored via `lib/core/*.js` etc.), and ship only inside the npm package (`files: ["lib/"]`) so parallix â€” which runs as plain Node CJS and `require()`s the compiled `.js` at runtime â€” keeps working. This supersedes the earlier "ship both / no cleanup step" idea, which caused autogenerated `.js` to be committed in TASK-1367.
- ES module syntax (`import`/`export`) with `module: NodeNext`
- Strict TypeScript checking enabled
- All tests must pass after each mission (baseline counts preserved)
- JSDoc-annotated codebase (283+ `@param`/`@returns` annotations) â€” leverage existing annotations

**Conversion pattern per file (learnings baked in from TASK-1366/1367):**
1. Rename `.js` â†’ `.ts` as a **faithful, minimal conversion** so `git diff -M` reports a rename **â‰Ą 50%** (history shows "moved then changed for ts fixes", not a rewrite). Preserve import names, helper functions, formatting, and comments; do not balloon JSDoc into verbose blocks.
2. Replace `require`/`module.exports` with `import`/`export`; preserve the exact public export shape (including any `_internals`). A minimal lazy `require()` is acceptable only to break a genuine circular dependency.
3. Add missing `@type`/inline type annotations where JSDoc is sparse
4. **`git rm --cached` the old compiled/source `.js`** so no autogenerated `.js` is tracked (the `lib/core/*.js` gitignore + `.eslintignore` globs keep regenerated artifacts out of git and out of lint).
5. Run `tsc --noEmit` to verify
6. Run `npm test` to verify no regression
7. Run `./scripts/verify-local.sh static-analysis` **with compiled `.js` present** to confirm the gate is green
8. Confirm distribution: `npm run prepublishOnly` regenerates the runtime `.js`, `npm pack --dry-run` lists them, and `require()` of each converted module loads cleanly.

**Shared infrastructure (landed in TASK-1367, available to all later waves):**
- `package.json` script `build:cjs` (`tsc --rootDir . --outDir . --module CommonJS --moduleResolution Node --esModuleInterop`), with `pretest` and `prepublishOnly` delegating to it. It compiles **every** `lib/**/*.ts` in place, so no per-wave `package.json` change is needed.
- `.gitignore` globs `lib/core/*.js`; `.eslintignore` globs `lib/core/*.js` with `!` negations for the still-hand-written files.
Later waves should assume they are rebased on top of this and reuse it rather than re-inventing it.

### Canonical "no-committed-`.js`" guardrails (apply in every wave)

For the directory `D` a wave converts:

1. **Faithful rename, not rewrite.** Each `X.js` â†’ `X.ts` must be minimal enough that `git diff -M --summary <merge-base> -- D/X.js D/X.ts` reports a `rename` **â‰Ą 50%**. Preserve import names, helper functions, formatting, blank lines, comments, and the exact public export shape (including any `_internals`); keep JSDoc terse. The reviewer must see "moved, then changed for ts fixes."
2. **Never commit compiled `.js`.** After converting `X.js`: `git rm --cached D/X.js` (and delete on disk). The artifact is regenerated by `npm run build:cjs`.
3. **Reconcile ignore files so they mirror reality (order-independent):**
   - `.gitignore`: ensure `D/*.js` is present (add once when the directory first enters conversion).
   - `.eslintignore`: ensure `D/*.js` is present **plus** a `!D/<file>.js` negation for every `.js` in `D` that is still hand-written (not yet converted). When you convert `D/X.js`, **delete its `!D/X.js` negation** so its compiled output is ignored, not linted.
   - This keeps the static-analysis gate green whether or not compiled artifacts are on disk, never bombs when a new `.ts` is added, and still lints genuine hand-written source.
4. **ESLint flat-config caveat (Mission 11 / TASK-1374):** once `.eslintrc.cjs` becomes `eslint.config.mjs`, `.eslintignore` is **no longer read** â€” every ignore/negation above must be ported into the flat config's `ignores` array (later entries override earlier, mirroring negation). Do not drop this coverage during the migration.
5. **A minimal lazy `require()`** is allowed only to break a genuine circular dependency.
6. **Verify with captured proof:** `tsc --noEmit` clean; `npm test` at baseline counts; `git ls-files D/*.js` lists only still-hand-written files; rename check (1) passes for each converted file; `grep -c 'module.exports' D/*.ts` is zero; `./scripts/verify-local.sh static-analysis` passes **with compiled `D/*.js` present on disk**; `npm run prepublishOnly && npm pack --dry-run | grep D/` shows the compiled `.js` shipped; `node -e "require('./D/X')"` loads with exports intact.
