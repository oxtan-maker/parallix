## Problem
`runDeclaredGates` (`lib/commands/handoff.js:~429`) executes each `## Gates` bullet verbatim via `bash -c`. When a gate is authored as prose (e.g. `npm run typecheck (tsc --noEmit) reports zero errors`) it either dies with a bash syntax error or silently passes the prose as junk args to a real command (`npm test passes with zero new failures` → `npm test` ignores the rest). This stranded task-1372 even though typecheck/tests were green.

## Fix options (pick during impl)
- Pre-validate each gate with `bash -n` (syntax check) at handoff start and fail fast with a clear, actionable message naming the offending gate and line.
- Optionally support a `description :: command` split so gates can carry human-readable intent without breaking execution.
- Add a lint/check that flags MISSION.md gates containing prose markers (parentheses, " reports ", " returns ", " passes ", " shows ") and surfaces them at mission authoring/refinement time, not handoff time.

## Acceptance
- A prose gate is rejected at handoff with a message identifying the gate and why, instead of an opaque bash syntax error.
- Existing valid command gates still run unchanged.
- Regression test covering a prose gate and a valid gate.
